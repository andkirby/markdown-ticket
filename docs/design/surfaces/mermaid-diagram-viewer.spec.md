# Mermaid Diagram Viewer

In-app overlay for inspecting rendered Mermaid diagrams with immediate pan and zoom.

## Composition

```text
MarkdownContent
└── MermaidContainer (.mermaid-container)
    ├── MermaidDiagram (.mermaid)
    │   └── RenderedSvg
    └── FullscreenButton (.mermaid-fullscreen-btn)

MermaidOverlayState (same MermaidContainer, data-overlay-enabled=true)
├── FullscreenButton (exit icon)
└── MermaidDiagram (.mermaid, transform target)
    └── RenderedSvg
```

## Children

| Child | Component | Spec | Conditional |
|-------|-----------|------|-------------|
| MarkdownContent | `src/components/MarkdownContent/` | — | always for markdown render |
| MermaidContainer | generated by `src/utils/mermaid/core.ts` | this file | only for Mermaid code blocks |
| FullscreenButton | created by `src/utils/mermaid/fullscreen.ts` | this file | after Mermaid render |
| MermaidDiagram | `.mermaid` element containing rendered SVG | this file | always inside container |

## Layout

### Inline State

- Mermaid container stays in normal markdown flow.
- Diagram markup remains `.mermaid-container > code.mermaid`.
- No extra pan/zoom wrapper is introduced.
- Fullscreen button sits at top-left of the Mermaid container.
- Inline diagram sizing must continue to fit the document width.
- Page scroll remains native.

### Overlay State

- Uses in-app overlay, not native browser `requestFullscreen()`.
- Same `.mermaid-container` becomes the overlay root.
- Overlay root:
  - `position: fixed`
  - `inset: 0`
  - `width: 100vw`
  - `height: 100vh`
  - `z-index: 9999`
  - `display: flex`
  - `align-items: center`
  - `justify-content: center`
  - `overflow: hidden`
- Body scroll is disabled while overlay is active and restored on exit.
- Diagram starts centered and scaled to fit within 90% of viewport width and height.
- Diagram aspect ratio is preserved.
- Movement happens through CSS transform on `.mermaid`.

## Interaction

| User Action | Required Behavior |
|-------------|-------------------|
| Click fullscreen button | Enter in-app overlay immediately |
| Click exit button while overlay open | Exit overlay and restore inline state |
| Escape while overlay open | Exit overlay only; do not close ticket detail or navigate away |
| Mouse wheel up / trackpad scroll up | Zoom in |
| Mouse wheel down / trackpad scroll down | Zoom out |
| Trackpad pinch with `ctrlKey` wheel events | Zoom in/out |
| Mouse down + drag on diagram | Pan diagram in drag direction |
| Mouse up outside diagram | End pan state |
| Single-finger touch drag | Pan diagram |
| Two-finger touch pinch | Zoom diagram |
| Double click / double tap | Reset to initial fit scale and centered pan |

## States

| State | Trigger | Visual / Behavior Change |
|-------|---------|--------------------------|
| inline | Markdown rendered | Diagram appears in document flow; button visible; native page scroll works |
| overlay-fit | Click fullscreen button | Container becomes fixed viewport overlay; body scroll locked; diagram scaled to fit |
| zooming | Wheel or pinch | Diagram scale changes within zoom limits; pan offset preserved |
| panning | Drag on diagram | Cursor changes to `grabbing`; diagram follows pointer |
| reset | Double click | Diagram returns to initial fit scale and pan `{ x: 0, y: 0 }` |
| exit | Exit button or Escape | Inline styles, zoom handlers, transform, display, cursor, and body scroll are cleared |

## Keyboard

| Key | Action |
|-----|--------|
| `Escape` | Close Mermaid overlay and stop propagation to surrounding ticket UI |

Focus trapping is not implemented in the current surface. If this viewer evolves into a modal shell, add `role="dialog"`, `aria-modal="true"`, focus trap, and focus return per `src/MODALS.md`.

## Responsive

| Breakpoint | Change |
|------------|--------|

| 640-1024px | Same layout; exit button remains top-left above diagram |

## Modal / Overlay Compliance

This surface intentionally differs from the standard modal pattern in `src/MODALS.md`:

- No `bg-black/50` backdrop, because the diagram surface itself becomes the inspection canvas.
- No click-outside close, because the whole viewport is the pan/zoom interaction surface.
- Uses `z-index: 9999` instead of `z-50` so the viewer can sit above ticket detail overlays.
- Escape and body scroll restoration are required and implemented.

If the viewer gets non-diagram content or separate controls beyond the exit button, revisit the standard modal requirements.

## Extension Notes

- Do not reintroduce a `.mermaid-panzoom` wrapper unless inline layout behavior is explicitly tested.
- Do not call native `requestFullscreen()` for this surface; it caused headed Chrome/macOS input delay.
- Keep pan/zoom dependency-free unless native handlers cannot satisfy the interaction contract.
- Add E2E coverage for immediate wheel/drag if browser automation can reproduce headed overlay behavior.
